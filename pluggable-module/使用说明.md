# 可插拔模块协议 — 使用说明

本文件夹是一套 **Cursor Agent Skill**，用来约束 AI：把任意功能（广告、登录、支付、推送等）抽成**可替换的独立模块**，而不是把代码揉进业务里。

---

## 文件夹内容

| 文件 | 作用 |
|------|------|
| `SKILL.md` | 主协议（AI 读取的核心约束） |
| `prompts.md` | 复制即用的提问模板 |
| `forbidden.md` | 禁止的耦合写法 |
| `checklist.md` | 交付前自检清单 |
| `integration-template.md` | 集成文件模板 |
| `使用说明.md` | 本文件 |

---

## 第一步：让 Cursor 能自动加载

已在你的电脑执行（若失效可手动重做）：

```bash
ln -sf ~/Desktop/pluggable-module ~/.cursor/skills/pluggable-module
```

完成后，所有项目都能通过 `@pluggable-module` 引用。

---

## 第二步：怎么跟 AI 提问

### 日常最短用法

```
按 pluggable-module 协议，把「登录」抽离为独立模块，先出方案等我确认。
```

### 引用 Skill

在 Cursor 对话里输入：

```
@pluggable-module
```

然后接着说你的具体需求。

### 三种场景对应话术

**场景 A — 从现有项目抽离功能**

```
@pluggable-module
把「支付」抽离为独立模块。先出架构方案、API 列表、文件清单，不要写代码。
```

**场景 B — 在新项目接入已有模块**

```
@pluggable-module
接入已有的 payment-core 模块。只允许改白名单文件，完成后跑验收五问。
```

**场景 C — 升级模块版本**

```
@pluggable-module
把 payment-core 从 1.2.0 升到 1.3.0。默认只改版本号，只看 BREAKING。
```

更多模板见 `prompts.md`。

---

## 第三步：你怎么验收 AI 的产出

让 AI 回答 **验收五问**（详见 `checklist.md`）：

1. 集成点文件数 = 1 ？
2. 业务没直接 import SDK ？
3. core 没 import app 包 ？
4. 换项目接入 ≤ 3 个文件 ？
5. 无 breaking 升版业务改动 = 0 ？

**有一项否 → 要求返工**，不要说「能跑就行」。

---

## 第四步：某个功能稳定后的进阶做法

当某个功能（如广告）协议稳定后，可复制本文件夹：

```bash
cp -r ~/Desktop/pluggable-module ~/.cursor/skills/ad-module
```

然后在 `ad-module/SKILL.md` 里补充：

- `ad-core` 的 Maven 坐标或模块路径
- 具体的 `AdFacade` API
- `changelog.md`

以后对新项目只说：

```
按 ad-module 接入，遵守 pluggable-module 母协议。
```

---

## 核心原则（记住这 5 句）

1. **单 Facade** — 业务只调一个入口
2. **单 Integration** — 全项目只有一个集成文件
3. **先方案后代码** — 不确认不写
4. **升级只换版本** — 无 breaking 不改业务
5. **Screen 像脚本** — 复杂逻辑进 module internal；禁止 app 层 Pipeline 藏流程

---

## 常见问题

**Q：AI 还是往 ViewModel 里写逻辑怎么办？**

把 `forbidden.md` 里对应条目贴回去，并说：「违反 pluggable-module 协议，按 checklist 返工。」

**Q：Desktop 上改了 Skill，Cursor 会同步吗？**

会。因为 `~/.cursor/skills/pluggable-module` 是指向桌面的软链接。

**Q：想给团队用？**

把整个 `pluggable-module` 文件夹拷到项目的 `.cursor/skills/` 下即可。
