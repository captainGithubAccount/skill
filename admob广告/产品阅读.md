## 你怎么告诉 AI 要加广告

复制 [广告位清单模板.md](广告位清单模板.md) 表格，或用自然语言说明：

- **哪个页面**（Activity / Fragment）
- **什么类型**（开屏 / 插屏 / 原生 / Banner）
- **什么时候展示**（进入页、点按钮、离开前）
- **原生/Banner 容器**（布局里的 ViewGroup id）
- **是否仅 B 面**

AI 会分配 `AdSense` 编号、改 JSON、写预加载与展示代码。

## 四种广告（产品视角）

| 类型 | 用户感受 | 无广告时 |
|------|----------|----------|
| 开屏 | 启动页全屏广告 | 直接进入下一页 |
| 插屏 | 全屏打断，关后继续 | 不弹窗，流程照常 |
| 原生 | 页面内一块广告区域 | 区域隐藏，不占空白 |
| Banner | 主页底部可折叠条 | 底部区域隐藏 |

**插屏 / 原生**：必须先预加载；没缓存就不展示，**不会**在点击时现场拉广告卡住。

**Banner**：进入主页时现场加载；切换 Tab 再回来一般复用已加载的广告。

## 开关与限制

| 条件 | 效果 |
|------|------|
| 订阅会员 | 全部不展示 |
| JSON 未配置该广告位 | 不请求、不展示 |
| UMP 流程未完成 | 全部不请求（与 SDK 初始化并列） |
| 日曝光 / 点击达上限 | 当日不再展示 |
| A 面 | 仅开屏 + enter/back 插屏（审核包） |
| B 面 | 语言页、Banner、底栏插屏、大原生等额外位 |
| B 面远程配置未拉到 | B 专属位仍不可用（沿用 A 本地配置） |

## 验收时看什么

| 你想确认 | 怎么做 |
|----------|--------|
| 广告有没有请求 | Logcat 搜 `TAG-->>AdRequest` + 场景名（如「主页Banner」） |
| 为什么没请求 | Logcat 搜 `TAG-->>vmodify` + `【广告位判定】` |
| 插屏点了没弹 | 看是否 `【展示跳过】原因=无可用缓存` |
| Banner 一直跳过 | A 面 / B 远程 `pdf_ad_config_b` 未生效 / ad_sense=4 缺失 |
| 原生空白条 | 应隐藏容器；有展示跳过正常 |
| 会员是否无广告 | 开订阅后各页无展示 |
| VPN 是否影响 | 关 VPN 后 FC 与广告应恢复 |
