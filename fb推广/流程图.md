# Meta（Facebook）App Events 推广 — 流程图

> 需求文档第七节定稿 · 树形简图  
> 仅 **App Events**，非 Audience Network 变现。

---

## 1. 总览：启动 → paid → Meta 三事件

```
冷启动 Application.onCreate
  │
  ├─ FacebookEventsDelegate.init(application)
  │    ├─ FacebookSdk.setApplicationId / setClientToken
  │    ├─ sdkInitialize + fullyInitialize
  │    ├─ AppEventsLogger.newLogger
  │    └─ AppEventsLogger.activateApp  → 安装归因 / 会话
  │
  ├─ InstallIdHolder.ensureFirstLaunchTimestamp（首次写入 KEY_FIRST_LAUNCH_APP_TS）
  │
  └─ AdEventUtils.facebookRevenueReporter = { reportFacebookPaidAdRevenue(revenue) }

用户观看广告且 AdMob 回调 OnPaidEventListener
  │
  └─ AdEventUtils.loadAdLoadedReport(adValue, responseInfo)
        ├─ Adjust / Firebase paid_ad_impression（本 skill 不展开）
        └─ facebookRevenueReporter(revenue, currencyCode)
              └─ FacebookEventsDelegate.reportPaidAdRevenue(revenue)
```

---

## 2. reportPaidAdRevenue(value) — 核心

```
reportPaidAdRevenue(value)
  │
  ├─ value <= 0  → 跳过，Log「收益<=0」
  │
  ├─ logAdImpressionRevenue(value)
  │    └─ logEvent(EVENT_NAME_AD_IMPRESSION, value + USD)
  │
  ├─ logAdRevenue(value)
  │    └─ logEvent("ad_revenue", value + USD)     → ROAS
  │
  └─ isInstallCalendarDay()?
         ├─ 是 → logAdD0Revenue(value)
         │         └─ logEvent("ad_d0_revenue", value + USD)
         └─ 否 → Log「非安装当日，跳过 ad_d0_revenue」
```

---

## 3. isInstallCalendarDay() — 安装自然日

```
isInstallCalendarDay()
  │
  ├─ MMKV KEY_FIRST_LAUNCH_APP_TS 未写入（<=0）
  │    └─ 视为安装当日 → true（首启边沿）
  │
  └─ 比较 Calendar(YEAR + DAY_OF_YEAR)
         ├─ 安装日 == 今天 → true
         └─ 否则 → false
```

`KEY_FIRST_LAUNCH_APP_TS` 在 **首次生成 install_id** 或 `obtainForBootstrap` 时写入，**只写一次**。

---

## 4. AdMob paid 接入点（须全覆盖）

```
开屏 / 插屏
  └── AdBridgeIntegration.revenueReporter.reportPaidEvent
        └── AdEventUtils.loadAdLoadedReport

原生
  └── NativeAdLoader.setOnPaidEventListener
        └── AdEventUtils.loadAdLoadedReport

（禁止）仅有 Firebase/Adjust paid、未调 loadAdLoadedReport → Meta 收不到收益
```

---

## 5. 安装日 vs 非安装日 — 事件条数速查

| 场景 | Ad_Impression | ad_revenue | ad_d0_revenue |
|------|---------------|------------|---------------|
| 安装自然日，value>0 | ✅ 每次 paid | ✅ 每次 paid | ✅ 每次 paid |
| 非安装日，value>0 | ✅ | ✅ | ❌ |
| value<=0 | ❌ 全跳过 | ❌ | ❌ |

## 6. 与其它模块边界

```
┌─────────────────────┐     ┌──────────────────────────┐
│ Meta App Events     │     │ Install Referrer         │
│ activateApp + 收入  │     │ AttributionManager       │
│ 事件                │     │ facebook_ads 渠道字符串  │
└─────────────────────┘     └──────────────────────────┘
         │                              │
         └─ Facebook 投放后台优化        └─ 客户端 A/B / 内部埋点
```

```
┌─────────────────────┐
│ ad_config_a / b     │  控制广告位是否请求（与 Meta 事件无 key 依赖）
└─────────────────────┘
```

---

## 7. 运营侧（代码外）

```
Facebook 开发者后台
  ├─ 应用编号 + Client Token 与 local.properties 一致
  └─ Events Manager
        ├─ 标准：Ad Impression（或 SDK 映射）
        ├─ 自定义：ad_d0_revenue
        └─ 自定义：ad_revenue（ROAS 优化对象）
```
