# AppsFlyer 接入 — 流程图

> vsave 定稿 · 树形简图  
> AF 应用内事件 **仅** `ad_impression` / `ad_click`；收入走 **logAdRevenue**。

---

## 1. 总览：启动 → 三类上报

```
冷启动 Application.onCreate
  │
  ├─ FirebaseApp.initializeApp
  ├─ KiteAnalytics.wire(installId, ...)
  └─ DeferredAttributionBridge.init(context, debug)
        └── AppsFlyerTracker.init
              ├─ Dev Key 为空 → 跳过，Log「未配置」
              └─ AppsFlyerLib.init + ConversionListener → isReady=true

进程进入前台（VsaveV3FeatureKit / Lifecycle）
  └── DeferredAttributionBridge.onAppForeground
        └── AppsFlyerTracker.startSession → AppsFlyerLib.start

用户看到广告
  ├── onAdImpression → AdTracker.trackImpression
  │     └── KiteAnalytics.emit(ad_impression) → AF + Firebase
  │
  ├── 用户点击 → AdTracker.trackClick
  │     └── KiteAnalytics.emit(ad_click) → AF + Firebase
  │
  └── onPaid（有收入时）
        └── AdEventUtils.loadAdLoadedReport
              ├── Adjust trackAdRevenue
              ├── AF logAdRevenue
              └── Firebase paid_ad_impression

FCM Token 刷新
  └── AppsFlyerFcmService.onNewToken
        └── AppsFlyerTracker.updateUninstallToken
```

## 2. SDK 初始化与归因

```
AppsFlyerTracker.init(context, debug)
  │
  ├─ isReady 已为 true → return
  ├─ APPSFLYER_DEV_KEY 为空 → Log 警告，return
  │
  └─ AppsFlyerLib.init(devKey, ConversionListener, app)
        ├─ debug → setDebugLog(true)
        ├─ onConversionDataSuccess → Log「归因转化数据」
        ├─ onConversionDataFail
        ├─ onAppOpenAttribution / onAttributionFailure
        └─ isReady = true，Log「init 完成 appId pkg」

AppsFlyerTracker.startSession(context)
  │
  ├─ !isReady → return
  └─ AppsFlyerLib.start → Log「AppsFlyer.start」
```

---

## 3. 应用内事件：仅 2 个走 AF

```
AdTracker.trackImpression(unitId, sense)
  └── TelemetryCatalog.ad_impression.emit(bundle)
        └── KiteAnalytics.emit("ad_impression", bundle)
              ├─ AppsFlyerTracker.logAdImpression(bundle)
              │     └── logEvent → ad_unit_id, ad_sense, ad_sense_label
              └─ FirebaseAnalytics.logEvent("ad_impression", bundle)

AdTracker.trackClick(unitId, sense, network?)
  └── bundle 含 ad_network（优先入参，否则 lastAdNetworkByUnitId）
        └── KiteAnalytics.emit("ad_click", bundle)
              ├─ AppsFlyerTracker.logAdClick
              └─ FirebaseAnalytics.logEvent

其它 KiteAnalytics.emit 事件名
  └── 仅 Firebase（guide_*、language_*、ump_* 等）
```

**不走 AF 的展示相关：**

```
AdBridgeIntegration.onAdImpression（全屏）
  └── DeferredAttributionBridge.emit("ad_show")
        └── Adjust + Firebase ad_show（无 AF 分支）
```

产品要的 AF 展示以 **AdTracker → ad_impression** 为准。

## 4. 广告收入 logAdRevenue

```
广告 load 成功
  └── AdjustHelper.openAd / interAd / nativeAd
        └── setOnPaidEventListener
              └── AdRevenueBridge.report(adValue, responseInfo)
                    └── AdBridgeIntegration.revenueReporter
                          └── AdEventUtils.loadAdLoadedReport
                                │
                                ├─ AdjustSdkUtil.trackAdMobPaidEvent
                                ├─ AppsFlyerTracker.logAdMobPaidEvent
                                │     ├─ !isReady → skip + Log
                                │     └─ AFAdRevenueData + logAdRevenue
                                │           monetizationNetwork = adSourceName
                                │           mediationNetwork = GOOGLE_ADMOB
                                ├─ AdTracker.rememberAdNetwork(adUnitId, network)
                                ├─ KiteAnalytics.emit("paid_ad_impression")
                                └─ facebookRevenueReporter?（Meta，本 skill 不展开）
```

---

## 5. 卸载衡量（FCM）

```
Firebase 下发新 Token
  └── AppsFlyerFcmService.onNewToken(token)
        └── AppsFlyerTracker.updateUninstallToken(ctx, token)
              ├─ !isReady 或 token 空 → return
              └─ AppsFlyerLib.updateServerUninstallToken
                    Log「updateServerUninstallToken ok」

前置条件
  ├── google-services.json
  ├── Manifest: MESSAGING_EVENT → AppsFlyerFcmService
  └── AF 控制台 Uninstall measurement 已开启
```

## 6. 平台分工一张表

| 数据类型 | AppsFlyer | Adjust | Firebase |
|----------|-----------|--------|----------|
| 安装归因 / start | ✅ init + start | ✅ init | Analytics 默认 |
| ad_impression | ✅ logEvent | ❌ | ✅ |
| ad_click | ✅ logEvent | ❌ | ✅ |
| ad_show（全屏回调） | ❌ | ✅ emit | ✅ |
| 广告收入 | ✅ logAdRevenue | ✅ trackAdRevenue | ✅ paid_ad_impression |
| ad_request 等 | ❌ | ❌ | ✅ |
| 业务埋点 | ❌ | 部分 token | ✅ |
