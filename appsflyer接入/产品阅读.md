## 已实现能力（vsave 定稿）

| 能力 | 说明 | 后台可见 |
|------|------|----------|
| SDK 初始化 | Dev Key + 包名；`local.properties` 配置 | 安装与归因 Overview |
| 基础归因 | 转化数据回调 + 进程前台 `start` | Attribution / 转化 Log |
| 应用内事件 | **仅** `ad_impression`、`ad_click` | Events |
| 广告收入 | AdMob **onPaid** → `logAdRevenue` | Ad Revenue / Monetization |
| 卸载衡量 | FCM `onNewToken` 上报 Token | Uninstall measurement |
| 独立工具类 | `com.isi.telemetry.AppsFlyerTracker` | — |

**明确不上报 AF 的内容：**

- `ad_request`、`ad_request_success`、`ad_request_fail` 等请求类事件
- `paid_ad_impression`（走 Firebase，AF 用专用 `logAdRevenue`）
- `DeferredAttributionBridge.emit("ad_show")` 等 Adjust 双路事件
- 引导页、语言页等业务 Firebase 事件

**三路并行（有 onPaid 时）：**

| 平台 | API / 事件 |
|------|------------|
| AppsFlyer | `logAdRevenue` |
| Adjust | `trackAdRevenue` |
| Firebase | `paid_ad_impression` |

## 表1：控制台核对表

| 你想确认的问题 | 去哪看 | 关键字 / 条件 |
|----------------|--------|----------------|
| AF 有没有收到广告收入？ | AppsFlyer → **Ad Revenue** / Monetization | `MediationNetwork = googleadmob`，或按 **Revenue** 趋势 |
| 收入按广告网络拆分？ | Ad Revenue → **Monetization network** | 对应 AdMob `adSourceName` |
| 展示次数对不对？ | AppsFlyer → **Events** → `ad_impression` | 参数 `ad_unit_id`、`ad_sense` |
| 点击次数对不对？ | Events → `ad_click` | 含 `ad_network` |
| 为什么 AF 没数据？ | Logcat 过滤 `AppsFlyerTracker` | `appsflyer.dev.key 未配置` / `未初始化` |
| 卸载衡量是否生效？ | AF → Uninstall measurement | Logcat：`updateServerUninstallToken ok`；需 FCM Token |
| 归因 / 安装从哪来？ | AF → Overview / Attribution | 转化回调 Log：`归因转化数据` |

### Logcat 自检（表1 补充）

| 标签 | 正常日志 |
|------|----------|
| `AppsFlyerTracker` | `AppsFlyer.init 完成` / `logAdRevenue ok` / `logEvent ok: ad_impression` |
| `AppsFlyerFcm` | `FCM onNewToken 已上报 AF 卸载衡量` |
| `TAG-->>isi_埋点` | 事件名 + 参数（Firebase 同发） |

## 表2：事件与字段配置表

### 2.1 应用内事件（仅 2 个，其余走 Firebase）

| 事件名 | 目的 | 触发时机 | 参数名 | 类型 | 示例 / 说明 |
|--------|------|----------|--------|------|-------------|
| `ad_impression` | 广告展示 | SDK `onAdImpression` 立即 | `ad_unit_id` | String | `ca-app-pub-xxx/6110` |
| | | | `ad_sense` | Int | `1`～`10`，与 `AdSense.id` 一致 |
| | | | `ad_sense_label` | String | 如 `语言选择页面原生`（便于后台查看） |
| `ad_click` | 广告点击 | 用户点击广告立即 | `ad_unit_id` | String | 同上 |
| | | | `ad_sense` | Int | 同上 |
| | | | `ad_sense_label` | String | 同上 |
| | | | `ad_network` | String | AdMob 回传网络名；**onPaid 后**缓存到同 unit 的点击 |

### 2.2 ad_sense 编号对照

| ad_sense | 含义 |
|----------|------|
| 1 | 加载冷启动开屏 |
| 2 | 语言选择页面原生 |
| 3 | 语言选择页面插屏 |
| 4 | 加载热启动开屏 |
| 5 | 主页原生 |
| 6 | 底部导航栏切换插屏 |
| 7 | 搜索页原生 |
| 8 | 搜索页插屏 |
| 9 | 引导大原生 |
| 10 | 引导插屏 |

### 2.3 广告收入（专用 API，不是 logEvent）

| API | 时机 | 主要字段 |
|-----|------|----------|
| `AppsFlyerLib.logAdRevenue` | 每次 AdMob **onPaid**（有收入） | `monetizationNetwork` = adSourceName；`mediationNetwork` = GOOGLE_ADMOB；`currency`；`revenue`；可选 `ad_unit`、`ad_type` |

## 本地配置（产品无需改代码时知晓）

```properties
appsflyer.app.id=你的_AF_应用ID
appsflyer.dev.key=你的_AF_Dev_Key
```

- Android 包名须与 AF 控制台应用一致（如 `com.ymhd.vsave`）
- 改 `local.properties` 后需 **Rebuild** 才会写入 APK

## 验收预期（不能保证后台即时有数的情况）

| 情况 | 说明 |
|------|------|
| 测试广告 | 常有展示/点击，**可能无** onPaid → Ad Revenue 为空属正常 |
| 网络 / AF 延迟 | 后台 Events 可能延迟数分钟～数小时 |
| 未展示广告 | VIP 或频控拦截时无 `ad_impression` |
| 点击早于付费 | `ad_click` 可能暂时缺 `ad_network` |
